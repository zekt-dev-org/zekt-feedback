# zekt-feedback
This repository is acting as the single point of entry, for opening issues, feature requests and act as a roadmap on behalf of the Zekt Services (web based parts as well as the corresponding zekt-action). Please tag your issue, in a structure way - or include "product/service" which having an issue / feature requested when submitting the github issue.

## Zekt components:

- the whole zekt solution is packaged into a mono-repo (consisting of logical components: infrastructure pipelines & deployment, public website, persistency layers & data-models, backend API's) - except the zekt-action which is residing in a separate public repo.
- Components to provide feedback (logically) are listed below:

pub_website - (https://www.zekt.dev)
management - (management.html console app)
backend_APIs - wide asortment of API's
data-model - persistency & data-models
